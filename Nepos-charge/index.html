Dos vistas,
un sistema.

03 · Stack e Integraciones

Qué conecta
con qué.

La app real necesita cuatro integraciones externas. En la demo están simuladas. En producción cada una tiene su API, sus credenciales y su manejo de errores.

Protocolo EV
OCPP 1.6
Open Charge Point Protocol
La "lengua" que habla el software con el hardware. Comunicación vía WebSocket entre el servidor Nepos y cada cargador físico. Hardware-agnostic: funciona con cualquier marca.
Fase 2
Pasarela de Pagos
PayPhone + Stripe
Ecuador-nativo + fallback global
PayPhone para tarjetas ecuatorianas y billetera móvil. Stripe como fallback internacional. Ambos soportan suscripciones, prepago y webhooks de confirmación.
Fase 3
Mapas
Google Maps API
Maps + Places + Directions
Mapa interactivo de estaciones, geolocalización del conductor, cálculo de distancia al cargador más cercano y navegación hasta la estación.
Fase 2
Base de Datos
PostgreSQL
Supabase · Cloud Run
Usuarios, sesiones, transacciones, estado de cargadores. Supabase como capa de acceso con autenticación integrada. Google Cloud Run para el backend.
Fase 2
Facturación
SRI Ecuador
Factura electrónica
Emisión automática de facturas electrónicas al finalizar cada sesión de carga. Cumplimiento normativo para operaciones comerciales en Ecuador.
Fase 3–4
CRM Nepos
API interna
AppSheet + Google Sheets
Los datos de uso de la red (sesiones, ingresos, usuarios) se sincronizan al CRM existente de Nepos vía API REST. Pipeline de clientes conectado al uso real.
Fase 4
Stack técnico
App Móvil
React Native
Expo · iOS + Android
Un solo codebase para ambas plataformas. La web app actual se convierte a React Native en la Fase 4.
Fase 4
Demo actual
HTML + JS
PWA · Netlify
La demo que estás viendo es una web app progresiva. Se puede instalar en cualquier teléfono sin pasar por las stores.
Listo
Backend
Node.js
Express · WebSocket · Docker
Servidor que maneja OCPP, autenticación, pagos y la API del panel. Desplegado en Google Cloud Run.
Fase 2
Panel operativo
React + Vite
Web privada · Sin stores
El panel de Nepos es una web app privada con login. No necesita estar en las stores — acceso directo desde el navegador.

04 · Inversión por Fase

Qué requiere
cada etapa.

El modelo de desarrollo es incremental: cada fase tiene entregables concretos y se puede pausar o acelerar según el avance del deal con los partners de infraestructura.

FASE 1
✓ Completo
Demo funcional
Tiempo: ~1 semana Herramientas: Claude + HTML/CSS/JS Infra: $0 (Netlify gratis) Entregable: demo deployada en URL pública
FASE 2
Backend OCPP + Base de datos
Tiempo: 2–3 semanas dev Google Cloud Run: ~$30–80/mes Supabase: gratis hasta escala Entregable: cargadores reales conectados
FASE 3
Pagos en producción
Tiempo: 1–2 semanas PayPhone: comisión por transacción Stripe: 2.9% + $0.30 por cobro Entregable: primer cobro real procesado
FASE 4
App nativa en stores
Tiempo: 3–5 semanas Apple Developer: $99/año Google Play: $25 único Google Maps API: ~$50–200/mes según uso Entregable: app en App Store + Play Store
FASE 5
Escala y franquicias
Tiempo: continuo Infra escala con la red La plataforma se convierte en activo propio Entregable: kit del franquiciado incluye la app

La decisión estratégica clave

Para la primera estación con BYD no es necesario estar en las stores. Una PWA (Progressive Web App) da el 90% de la experiencia nativa y se puede tener lista en días. Las stores se encargan cuando hay tracción real de usuarios — típicamente a partir de la Fase 3 cuando los pagos están funcionando y hay conductores reales usando el sistema.
